The Keyframe API and SDKs provide simple interfaces for adding personas to AI agents and applications. This guide will walk you through the process of setting up a minimal example of a persona that works with your favorite agent or LLM. By the end, you’ll have a working persona that you can video call with in your web browser.

## Prerequisites

- [Sign up](platform.keyframelabs.com) to the Keyframe Labs platform and [create a new API key](/guides/getting-started/get-your-api-key)
- An agent framework or real-time LLM to serve as your "provider"

### Prepare your agent or LLM provider details

<Tabs>
  <Tab title="ElevenLabs Agents">
    To use ElevenLabs Agents as your agent provider, you'll need:

    - The agent ID from ElevenLabs of the agent you wish to use
    - [A signed URL from ElevenLabs for the chosen agent](https://elevenlabs.io/docs/agents-platform/customization/authentication#using-signed-urls)

    And then we'll use this snippet in the steps to follow:

    ```typescript
    type VoiceAgentDetails = {
      type: 'elevenlabs';
      agent_id?: '<your_eleven_agent_id>';
      signed_url?: '<your_eleven_agent_signed_url>';
    };
    ```
  </Tab>
  <Tab title="LiveKit Agents">
    <Info>
      For LiveKit Agents, use the [LiveKit x Keyframe plugin](/guides/self-managed/agent-frameworks/livekit-agents) instead of the guide below.
    </Info>
  </Tab>
  <Tab title="OpenAI Realtime">
    To use OpenAI Realtime as your real-time LLM provider, you'll need:

    - [An ephemeral token](https://developers.openai.com/api/docs/guides/realtime-webrtc#creating-an-ephemeral-token) for your desired Realtime session configuration

    ```typescript
    type VoiceAgentDetails = {
      type: 'openai';
      token?: string;
      system_prompt?: string;
    };
    ```
  </Tab>
</Tabs>
