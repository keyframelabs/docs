# Which integration approach should I choose?

There are 3 ways to use Keyframe's interactive video models:

1. **Widget:** Keyframe manages everything. Drop a snippet into your application and go live in minutes with a configurable, no-code widget. Best if you're starting from scratch.
2. **Hosted:** Bring your own agent or LLM. Share your agent or LLM credentials with Keyframe's platform to get a configurable, low-code embed with an optimized speech-to-video experience. Best if you have an existing voice AI in production and want to add a Keyframe persona without building UX and plumbing from scratch.
3. **Self Managed:** Bring your own agent or LLM and keep your API keys on your own infrastructure. Build a fully custom embed using the same JS primitives that power our Widget and Hosted approaches. Best if you want maximum control.

| **Feature** | Widget | Hosted | Self Managed |
| --- | --- | --- | --- |
| Bring your own agent or LLM | ❌ | ✅ | ✅ |
| Share agent or LLM credentials with Keyframe | ❌ | ✅ | ❌ |
| Keyframe managed agent or LLM | ✅ | ❌ | ❌ |
| Configurable no-code widget | ✅ | ❌ | ❌ |
| Configurable low-code embed | ❌ | ✅ | ❌ |
| Custom embed via Keyframe's JS primitives | ❌ | ✅ | ✅ |
| Per session avatar selection | ❌ | ❌ | ✅ |

# Self managed

For the self managed pipeline, Keyframe personas integrate with both agent frameworks (e.g., Cartesia, ElevenLabs, LiveKit, Vapi) and real-time LLMs (e.g., Gemini Live, OpenAI Realtime). While each integration has minor setup differences, the overall flow is largely the same.

## Pipeline overview

<Steps>
  <Step title="Bring your own agent or LLM">
    Your client application continues to interface with your existing agent framework or real-time LLM
  </Step>
  <Step title="Keyframe no-code or low-code components">
    Our easy-to-use components recieve "raw" audio from your agent or LLM, and relays it to the Keyframe platform
  </Step>
  <Step title="Rendered persona">
    The Keyframe components also manage rendering the generated persona in your client application
  </Step>
</Steps>

```text
+---------------------+                                     +--------------------------+
|                     | ----------- User input -----------> |                          |
|                     |                                     |    Your existing agent   |
|                     | <----- Raw agent / LLM audio ------ |         or LLM           |
|                     |       (will now be "relayed")       |                          |
|                     |                                     +--------------------------+
|   Your client app   |
|                     |                                     +--------------------------+
|                     | ---- Relayed agent / LLM audio ---> |                          |
|                     |                                     |         Keyframe         |
|                     | <------- Rendered persona --------- |         platform         |
|                     |    (w/ synced audio and video)      |                          |
+---------------------+                                     +--------------------------+
```
