## Which integration approach should I choose?

There are 3 ways to integrate Keyframe's interactive video models:

1. **Widget:** Design your agent or LLM directly on the Keyframe platform with a supported provider, then drop a snippet into your application to go live in minutes with a configurable, no-code widget. Best if you don't want to write any code, or are starting from scratch (i.e., no existing agent or LLM).
2. **Hosted:** Bring your own agent or LLM. Share your agent or LLM credentials with Keyframe's platform to get a configurable, low-code embed with an optimized speech-to-video experience. Best if you have an existing agent or LLM in production and want to add a Keyframe persona without building UX and common plumbing from scratch.
3. **Self Managed:** Bring your own agent or LLM and keep your API keys on your own infrastructure. Build a fully custom embed using the same JS primitives that power our Widget and Hosted approaches. Best if you want maximum control.

| **Feature** | Widget | Hosted | Self Managed |
| --- | --- | --- | --- |
| Design agent or LLM on Keyframe platform | ✅ | ❌ | ❌ |
| Bring your own agent or LLM | ❌ | ✅ | ✅ |
| Share agent or LLM credentials with Keyframe | ❌ | ✅ | ❌ |
| Keyframe managed agent or LLM | ✅ | ❌ | ❌ |
| Configurable no-code widget | ✅ | ❌ | ❌ |
| Configurable low-code embed | ❌ | ✅ | ❌ |
| Custom embed via Keyframe's JS primitives | ❌ | ✅ | ✅ |
| Per session avatar selection | ❌ | ❌ | ✅ |
